# 阶段 1：需求文档生成

## 目标
请基于当前项目目标，帮助我完成需求文档。

## 输入
- 当前项目根目录为本仓库根目录
- 已有资料、资源、代码、样例数据由我在当前会话中补充
- 若仓库内已有相关文件，请优先读取并引用

## 输出
- 生成或更新：`docs/proposal.md`

## 步骤
1. 先理解项目目标、范围、约束、输入资源、预期输出。
2. 如果我对领域不熟，请使用**提问**的方式帮助我澄清需求。
3. **不要猜测我的意图**。任何会影响需求边界、验收标准、非目标、技术限制的重要问题，都必须先向我提问。
4. 需求文档至少应包含：
   - 背景与目标
   - 用户故事 / 使用场景
   - 范围（In Scope / Out of Scope）
   - 功能需求
   - 非功能需求
   - 输入 / 输出
   - 约束条件
   - 验收标准
5. 输出写入 `docs/proposal.md`。

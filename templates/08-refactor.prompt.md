# 阶段 8：重构 Prompt

## 目标
在不改变外部行为或在明确控制行为变化的前提下，对现有实现做结构优化。

## 输入
- `docs/detailed-design.md`
- 相关模块任务文件
- 若存在，读取 `docs/changes/refactor-<name>.md`

## 输出
- 更新后的代码
- 必要时更新设计文档和任务文件
- 保持或补充测试
- 更新 `progress.md`

## 步骤
1. 先明确本次重构的范围、目标和非目标。
2. 优先保持外部接口与行为稳定；若确需变化，先记录到变更文件。
3. 先补保护性测试，再做重构。
4. 重构时避免同时引入无关功能。
5. 完成后运行完整检查。

# 完成定义

一个任务被标记为完成，至少应满足：

1. 功能已实现，且与当前任务描述一致
2. 对应测试已添加或更新
3. `lint` / `typecheck` / `test` 通过
4. 若改动影响接口、约束、假设或模块边界，相关文档已更新
5. 对应的任务文件 checklist 已更新
6. `docs/tasks/progress.md` 已同步

## 额外约束

- 不允许仅因“代码可运行”就标记完成
- 不允许用跳过测试或关闭检查来换取通过
- 不允许在未记录的前提下偷偷扩大任务范围

## 建议补充

对关键模块，除了单元测试，还应至少具备以下之一：

- 集成测试
- 端到端验收测试
- 黄金样例 / 基准输入输出测试
